# RBAC, Scope of Practice & HIPAA Research

> Compliance-critical research for role-based access control in ABA therapy practice management software. Covers BACB scope of practice by certification level, HIPAA minimum necessary standard, payer audit requirements, and day-to-day workflows for 1-50 staff practices.

---

## 1. RBT Scope & Restrictions

### What RBTs Can Do (Under Supervision)

RBTs are **paraprofessional** certificants who practice under the **direction and close supervision** of an RBT Supervisor (BCBA/BCBA-D) and/or RBT Requirements Coordinator. The supervisor is **responsible for all work the RBT performs**.

| Activity                                               | Permitted?  | Notes                                                                                             |
| ------------------------------------------------------ | ----------- | ------------------------------------------------------------------------------------------------- |
| Implement behavior intervention plans (BIPs)           | Yes         | Only plans designed by supervising BCBA                                                           |
| Collect behavioral data during sessions                | Yes         | Core responsibility                                                                               |
| Write session notes                                    | Yes         | Must be reviewed/co-signed by BCBA within 48 hours                                                |
| Implement skill acquisition programs                   | Yes         | Following BCBA-designed protocols                                                                 |
| Use punishment/restrictive procedures                  | Conditional | Only after supervisor verifies competence AND procedures are documented in a behavior-change plan |
| Modify treatment plans                                 | **No**      | Requires BCBA approval for any modification                                                       |
| Conduct assessments (FBA, VB-MAPP, etc.)               | **No**      | Assessment is BCBA scope only                                                                     |
| Develop behavior intervention plans                    | **No**      | BCBA scope only                                                                                   |
| Bill insurance independently                           | **No**      | Services billed under supervising BCBA's NPI (some payers require RBT's own NPI with modifiers)   |
| Discuss confidential info without supervisor direction | **No**      | Can only share under supervisor's direction                                                       |
| Make clinical decisions                                | **No**      | Must follow supervisor's direction                                                                |

**Source:** [BACB RBT Ethics Code 2.0](https://www.bacb.com/wp-content/uploads/2022/01/RBT-Ethics-Code-240830-a.pdf); [BACB RBT Task List 2nd Edition](https://www.bacb.com/wp-content/uploads/2020/05/RBT-2nd-Edition-Task-List_240830-a.pdf)

### RBT Confidentiality Requirements (RBT Ethics Code 2.0)

- RBTs **protect the confidentiality and privacy of their clients**
- RBTs only use **relevant identifying information** in job-related communications
- RBTs only discuss confidential client information **under the direction of their supervisor**, unless allowed by law (e.g., protecting from harm)
- RBTs **do not share** identifying information (photos, videos, written info) about clients on social media or websites
- RBTs must **document all actions** taken in response to legal or ethical issues

**Source:** [BACB RBT Ethics Code 2.0](https://www.bacb.com/wp-content/uploads/2022/01/RBT-Ethics-Code-240830-a.pdf); [Relias - RBT Code of Ethics](https://www.relias.com/blog/rbt-code-of-ethics)

### What Data RBTs Need Access To

During sessions, RBTs need:

- **Assigned client's** treatment goals, targets, and protocols
- **Behavior Intervention Plan (BIP)** for replacement behaviors and strategies
- **Data sheets** (labeled with client name, date, session start time)
- **Program instructions** for skill acquisition
- **Reinforcement preferences/items** list
- **Session schedule** (their own)

**Source:** [Artemis ABA - RBT Software Tools](https://www.artemisaba.com/rbt); [ATCC - Data Collection Methods for RBTs](https://www.atcconline.com/blog/data-collection-aba-fun-simple-behavior-tracking-pro); [Praxis Notes - RBT Task List A-01](https://www.praxisnotes.com/resources/rbt-tasklist-a01-prepare-data-collection)

### What Data RBTs Should NOT See

| Data Category                                                                 | Rationale                                          |
| ----------------------------------------------------------------------------- | -------------------------------------------------- |
| Other clients' records (not assigned to them)                                 | HIPAA minimum necessary; no treatment relationship |
| Insurance/billing details (rates, claim status, payer info)                   | Outside scope of practice; no job function need    |
| Authorization financials (approved units dollar values)                       | Billing is BCBA/billing staff responsibility       |
| Other RBTs' schedules or performance data                                     | No supervisory authority                           |
| Practice financial data (revenue, AR, payroll)                                | Administrative data; no job function need          |
| Provider credentials and HR files                                             | Administrative data                                |
| Full client demographics beyond what's needed for session (SSN, insurance ID) | HIPAA minimum necessary                            |

### Software Access Implications for RBT Role

- **See:** Own schedule, assigned clients only, session data entry, BIP/treatment goals for assigned clients, own session notes
- **Do:** Enter session data, write session notes, clock in/out, view upcoming schedule
- **Cannot:** Access unassigned clients, view billing data, modify treatment plans, approve sessions, view practice-wide reports, export data

## 2. BCaBA Scope & Restrictions

### What BCaBAs Can Do (Under BCBA Supervision)

BCaBAs are **mid-level** certificants who operate **under the supervision of a BCBA or BCBA-D**. They cannot practice independently. The BACB requires ongoing supervision for BCaBAs, with at least **5% of behavior-analytic service hours** supervised.

| Activity                                                    | Permitted? | Notes                                                                         |
| ----------------------------------------------------------- | ---------- | ----------------------------------------------------------------------------- |
| Assist with assessments                                     | Yes        | Under BCBA direction                                                          |
| Implement behavior intervention plans                       | Yes        | Designed by supervising BCBA                                                  |
| Collect and analyze data                                    | Yes        | Core responsibility                                                           |
| Supervise RBTs                                              | Yes        | BCaBAs can serve as RBT supervisors                                           |
| Conduct independent assessments (FBA)                       | **No**     | Cannot independently assess; must be under BCBA supervision                   |
| Independently create/modify treatment plans                 | **No**     | Must obtain BCBA approval for any modifications                               |
| Practice independently without BCBA oversight               | **No**     | BACB requires ongoing supervision                                             |
| Serve as responsible certificant for an organization        | **No**     | Since November 2019, only BCBAs/BCBA-Ds may serve as responsible certificants |
| Make independent clinical decisions about treatment changes | **No**     | Requires BCBA approval                                                        |

**Source:** [BACB BCaBA Handbook](https://www.bacb.com/wp-content/uploads/2025/08/BCaBAHandbook_250820-a.pdf); [BACB Supervision & Training](https://www.bacb.com/supervision-and-training/); [Yellow Bus ABA - BCBA vs BCaBA](https://www.yellowbusaba.com/post/bcba-vs-bcaba); [IntelliStars - BCBA vs BCaBA](https://www.intellistarsaba.com/blog/bcba-vs-bcaba)

### BCaBA Confidentiality Requirements

BCaBAs are bound by the same **Ethics Code for Behavior Analysts** as BCBAs (Sections 2.03-2.05). They must:

- Protect confidentiality of clients, stakeholders, supervisees, and trainees
- Only share confidential information with informed consent, to protect from harm, to resolve contractual issues, to prevent crime, or when compelled by law
- Comply with all documentation storage, retention, and destruction requirements

**Source:** [BACB Ethics Code for Behavior Analysts](https://www.bacb.com/wp-content/uploads/2022/01/Ethics-Code-for-Behavior-Analysts-240830-a.pdf)

### What Data BCaBAs Need Access To

- **Assigned clients'** full clinical records (treatment plans, BIPs, assessment data)
- **RBT session data** for clients they supervise
- **Supervision documentation** for RBTs they oversee
- **Their own schedule** and supervision schedule
- **Authorization status** for assigned clients (units remaining, dates) to coordinate services
- **Client demographics** necessary for clinical service delivery

### What Data BCaBAs Should NOT See

| Data Category                                              | Rationale                              |
| ---------------------------------------------------------- | -------------------------------------- |
| Clients not assigned to them                               | HIPAA minimum necessary                |
| Practice-wide financial/billing data                       | Outside scope; administrative function |
| Insurance claim details and payment amounts                | Billing staff responsibility           |
| Other BCaBAs'/BCBAs' supervision records (unless assigned) | No supervisory relationship            |
| Practice HR data, payroll, revenue                         | Administrative function                |
| Billing rates and fee schedules                            | Financial/administrative data          |

### Software Access Implications for BCaBA Role

- **See:** Assigned clients' full clinical records, RBT session data for supervised RBTs, own schedule, authorization utilization for assigned clients, supervision logs
- **Do:** Enter clinical notes, review/co-sign RBT session notes, document supervision, create data summaries, view client progress
- **Cannot:** Independently create assessments or treatment plans (needs BCBA approval workflow), access unassigned clients, view billing/financial data, modify authorizations, access practice-wide admin data

## 3. BCBA/BCBA-D Scope & Restrictions

### What BCBAs Can Do

BCBAs are **independent practitioners** who provide behavior-analytic services. BCBA-D adds a doctoral-level distinction but does not change scope of practice.

| Activity                                               | Permitted? | Notes                                   |
| ------------------------------------------------------ | ---------- | --------------------------------------- |
| Conduct comprehensive assessments (FBA, VB-MAPP, etc.) | Yes        | Core competency                         |
| Develop treatment/behavior intervention plans          | Yes        | Core competency                         |
| Supervise BCaBAs, RBTs, and other staff                | Yes        | Must meet BACB supervision requirements |
| Make clinical decisions independently                  | Yes        | Independent practitioner status         |
| Bill insurance directly                                | Yes        | Under their own NPI                     |
| Modify treatment plans                                 | Yes        | Based on data analysis                  |
| Sign off on RBT/BCaBA session notes                    | Yes        | Required within 48 hours for RBT notes  |
| Serve as organization's responsible certificant        | Yes        | BCBAs/BCBA-Ds only (since Nov 2019)     |
| Request and manage authorizations                      | Yes        | Clinical justification for services     |
| Discontinue or transition services                     | Yes        | With documented rationale               |

### BCBA Ethics Requirements for Billing & Documentation

**Section 2.06 - Accuracy in Service Billing and Reporting:**

> "Behavior analysts identify their services accurately and include all required information on reports, bills, invoices, requests for reimbursement, and receipts."

**Section 3.05 - Financial Agreements:**

> "Before beginning services, behavior analysts document agreed-upon compensation and billing practices with their clients, relevant stakeholders, and/or funders."

**Section 3.11 - Documenting Professional Activity:**

> "Throughout the service relationship, behavior analysts create and maintain detailed and high-quality documentation of their professional activities."

**Section 2.05 - Documentation Protection and Retention:**

> "Behavior analysts are knowledgeable about and comply with all applicable requirements for storing, transporting, retaining, and destroying physical and electronic documentation."

**Section 4.05 - Maintaining Supervision Documentation:**

> "Behavior analysts create, update, store, and dispose of documentation related to their supervisees or trainees by following all applicable requirements. They ensure their documentation is accurate and complete... and retain supervision documentation for at least 7 years."

**Section 4.09 - Delegation of Tasks:**

> "Behavior analysts delegate tasks to their supervisees or trainees only after confirming that they can competently perform the tasks and that the delegation complies with applicable requirements."

**Source:** [BACB Ethics Code for Behavior Analysts (readkong)](https://www.readkong.com/page/ethics-code-for-behavior-analysts-bacb-6956173); [BACB Ethics Codes](https://www.bacb.com/ethics-information/ethics-codes/)

### What Data BCBAs Need Access To

- **All assigned clients'** complete clinical and administrative records
- **All supervised staff's** session notes and supervision documentation
- **Authorization status and utilization** for assigned clients
- **Billing information** related to their services (what was billed, under their NPI)
- **Client demographics and insurance information** needed for service delivery
- **Practice-wide clinical quality data** (for Clinical Directors / lead BCBAs)

### What Data BCBAs Should NOT See (Unless in Admin/Director Role)

| Data Category                                                       | Rationale                                          |
| ------------------------------------------------------------------- | -------------------------------------------------- |
| Clients not assigned to them (unless Clinical Director)             | HIPAA minimum necessary                            |
| Detailed financial data (practice revenue, AR aging, profit/loss)   | Administrative function unless also owner/director |
| Payroll and HR data                                                 | Administrative function                            |
| Other BCBAs' clients' full records (unless clinical oversight role) | HIPAA minimum necessary                            |

### Software Access Implications for BCBA Role

- **See:** Assigned clients' full records (clinical + auth + demographics), supervised staff's documentation, authorization utilization, own billing history, own schedule
- **Do:** Create assessments, develop treatment plans, write clinical notes, review/co-sign all supervised staff notes, manage authorizations (request, track utilization), create supervision documentation, run clinical reports for assigned caseload
- **BCBA + Clinical Director additional access:** All clients across practice, all clinical staff's notes, practice-wide clinical metrics, quality assurance data

## 4. Billing Staff

### Billing Coordinator/Specialist Role in ABA

Billing staff in ABA practices are responsible for managing the revenue cycle. They are often **not clinically certified** (no BCBA/RBT) but need healthcare billing knowledge.

**Core responsibilities:**

- Process and submit insurance claims
- Verify client eligibility and benefits
- Track and obtain prior authorizations
- Post payments and manage accounts receivable
- Resolve claim denials and discrepancies
- Handle credentialing/re-credentialing for providers
- Monitor authorization utilization (units remaining, expiration dates)
- Generate financial reports (AR aging, collections, denial rates)
- Ensure billing code accuracy (CPT codes: 97151-97158, 0362T, 0373T)

**Source:** [ZipRecruiter - ABA Billing Jobs](https://www.ziprecruiter.com/Jobs/Aba-Billing); [Indeed - ABA Billing Specialist Jobs](https://www.indeed.com/q-aba-therapy-billing-specialist-jobs.html)

### What Data Billing Staff Need Access To

| Data Category                                                               | Why                                              |
| --------------------------------------------------------------------------- | ------------------------------------------------ |
| Client demographics (name, DOB, address)                                    | Required on claims                               |
| Insurance information (payer, member ID, group #, policy dates)             | Required for claims and eligibility verification |
| Authorization details (auth number, approved units, date ranges, CPT codes) | Required for claims and utilization tracking     |
| Session dates, start/end times, CPT codes billed, rendering provider        | Required for claim submission                    |
| Provider credentials (NPI, license #, BCBA certification)                   | Required for credentialing and claims            |
| Claim status and payment history                                            | Core job function                                |
| Denial reasons and correspondence                                           | Core job function for denial management          |
| Fee schedules and contracted rates                                          | Required for billing accuracy                    |

### What Data Billing Staff Should NOT See

| Data Category                                                                                           | Rationale                                                                        |
| ------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------- |
| Clinical session note narrative content (behavioral descriptions, interventions, clinical observations) | HIPAA minimum necessary - billing needs codes and times, not clinical narrative  |
| Assessment reports (FBA, VB-MAPP results, diagnostic details)                                           | Clinical data not needed for billing (unless required to justify a claim appeal) |
| Detailed behavior data (trial data, frequency counts, graphs)                                           | Clinical data; no billing function                                               |
| Treatment plan clinical details (specific goals, behavioral definitions)                                | Clinical data beyond what's needed for authorization matching                    |
| Progress notes narrative content                                                                        | Clinical data not needed for routine billing                                     |

**Critical nuance:** Billing staff **may** need limited clinical information when:

- Appealing a denial (may need clinical justification excerpts)
- Requesting prior authorization (may need diagnosis codes and clinical summary)
- Responding to a payer audit (may need to locate and produce clinical records, but not interpret them)

In these cases, access should be **temporary or request-based**, not standing access to full clinical records.

**Source:** [Secureframe - HIPAA Minimum Necessary Rule](https://secureframe.com/hub/hipaa/minimum-necessary-rule); [AccountableHQ - Minimum Necessary Standard](https://www.accountablehq.com/post/hipaa-s-minimum-necessary-standard-what-it-requires-of-healthcare-workers)

### Software Access Implications for Billing Staff

- **See:** Client demographics, insurance info, authorization details (units, dates, CPT codes), session metadata (date, time, CPT code, provider, units), claim status, payment data, provider credentials, fee schedules, financial reports
- **Do:** Submit claims, post payments, manage denials, verify eligibility, track authorizations, generate billing reports, manage provider credentialing
- **Cannot:** Access full clinical notes, view behavior data, modify treatment plans, access clinical assessments, view session narrative content

## 5. HIPAA Minimum Necessary Standard

### Core Requirement

The HIPAA Minimum Necessary Standard (45 CFR 164.502(b), 164.514(d)) requires covered entities to **limit the use, disclosure, and access to PHI to the least amount necessary to accomplish the intended purpose**.

> "Covered entities are required to make reasonable efforts to ensure that uses and disclosures of PHI are limited to the minimum necessary information to accomplish the intended purpose."

**Source:** [HHS - Summary of HIPAA Privacy Rule](https://www.hhs.gov/hipaa/for-professionals/privacy/laws-regulations/index.html); [Secureframe - HIPAA Minimum Necessary Rule](https://secureframe.com/hub/hipaa/minimum-necessary-rule)

### How It Applies to Each ABA Practice Role

| Role                              | PHI Access Level                                                                                                   | Justification                                           |
| --------------------------------- | ------------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------- |
| **RBT**                           | Assigned clients only: treatment goals, BIP, data collection forms, session schedule                               | Direct service delivery for assigned clients only       |
| **BCaBA**                         | Assigned clients' clinical records + supervised RBTs' session data                                                 | Clinical service delivery + supervision duties          |
| **BCBA**                          | Assigned clients' full records (clinical + admin) + supervised staff documentation                                 | Independent practitioner + supervision responsibilities |
| **BCBA (Clinical Director)**      | All clients' clinical data, all staff's clinical documentation                                                     | Clinical oversight of entire practice                   |
| **Billing Staff**                 | All clients' demographics, insurance, auth details, session metadata (dates/times/codes) — NOT clinical narratives | Payment processing and revenue cycle management         |
| **Office Manager/Admin**          | Client demographics, scheduling data, staff schedules, insurance info for verification — NOT clinical data         | Practice operations management                          |
| **Practice Owner (non-clinical)** | Aggregate/de-identified data preferred; limited PHI as needed for operations                                       | Business operations; prefer de-identified when possible |

### Implementation Requirements for Software

Organizations must develop **role-based access control (RBAC)** policies that:

1. **Identify roles** requiring PHI access
2. **Categorize which PHI types** each role needs
3. **Specify conditions** for access
4. **Maintain RBAC matrices** documenting role-permission mappings
5. **Maintain audit logs** tracking all access attempts
6. **Implement automated alerts** for suspicious activity (repeated failed logins, unusual access volumes, off-hours activity)
7. **Support "break-the-glass"** emergency access with reason codes and audit trails

**Source:** [Censinet - HIPAA Access Control Requirements](https://censinet.com/perspectives/hipaa-access-control-requirements-explained); [AccountableHQ - Minimum Necessary Standard for Healthcare Workers](https://www.accountablehq.com/post/hipaa-s-minimum-necessary-standard-what-it-requires-of-healthcare-workers)

### Exceptions to Minimum Necessary

The minimum necessary standard does **NOT apply** to:

- Disclosures to or requests by a healthcare provider for **treatment** purposes
- Disclosures to the individual who is the subject of the information
- Uses or disclosures authorized by the individual
- Disclosures to HHS for compliance investigations
- Uses or disclosures required by law

This means clinicians providing treatment (BCBAs, BCaBAs, RBTs in a treatment context) have broader access to clinical information for their assigned clients than non-treatment staff.

## 6. HIPAA Technical Safeguards & Audit Requirements

### Access Control (45 CFR 164.312(a))

| Control                         | Status       | Requirement                                                                                                        |
| ------------------------------- | ------------ | ------------------------------------------------------------------------------------------------------------------ |
| **Unique User Identification**  | **Required** | Each user must have a unique ID. Shared accounts prohibited. Logs must capture user ID, date/time, device, actions |
| **Emergency Access Procedures** | **Required** | Documented "break-the-glass" procedures for emergency ePHI access                                                  |
| **Automatic Logoff**            | Addressable  | Systems should auto-lock after inactivity (5-15 min typical). Sensitive actions may require re-authentication      |
| **Encryption/Decryption**       | Addressable  | At rest and in transit. If not implemented, must document alternative safeguards                                   |

### Audit Controls (45 CFR 164.312(b))

**Required.** Organizations must implement mechanisms that **record and examine** all activity in systems containing ePHI.

**What must be logged:**

- **User ID** performing the action
- **Date and time** of access
- **Event type** (view, create, update, delete, export, print)
- **Patient/record identifier** accessed
- **Source IP/device**
- **Event outcome** (success/failure)

**Three categories of audit trails:**

1. **User Audit Trail:** Log-on events, authentication attempts, password changes
2. **System Audit Trail:** Device identification, IP addresses, firewall status
3. **Application Audit Trail:** File access, record creation/revision/deletion, data export

**Retention:** Audit logs must be retained for **at least 6 years** per 45 CFR 164.316(b)(2)(i).

**Source:** [eCFR - 45 CFR 164.312](https://www.ecfr.gov/current/title-45/subtitle-A/subchapter-C/part-164/subpart-C/section-164.312); [Cornell LII - 45 CFR 164.312](https://www.law.cornell.edu/cfr/text/45/164.312); [Optro - HIPAA Audit Trail Requirements](https://optro.ai/blog/hipaa-audit-trail-requirements); [Pangea - HIPAA Audit Log Requirements](https://pangea.cloud/blog/hipaa-audit-log-requirements/)

### Workforce Security (45 CFR 164.308(a)(3))

| Control                       | Status      | Requirement                                                          |
| ----------------------------- | ----------- | -------------------------------------------------------------------- |
| **Authorization/Supervision** | Addressable | Procedures for authorizing workforce member access to ePHI           |
| **Workforce Clearance**       | Addressable | Determine that access is appropriate for the workforce member's role |
| **Termination Procedures**    | Addressable | Terminate ePHI access when employment ends or role changes           |

**"Addressable" does not mean optional** - organizations must implement the control, implement an equivalent alternative, or document why it's not reasonable.

**Source:** [AccountableHQ - 45 CFR 164.308 Explained](https://www.accountablehq.com/post/45-cfr-164-308-explained-the-full-hipaa-administrative-safeguards-requirements-compliance-guide); [Bricker Graydon - Administrative Safeguards](https://www.brickergraydon.com/insights/resources/key/hipaa-security-regulations-administrative-safeguards-164-308)

### Person/Entity Authentication (45 CFR 164.312(d))

**Required.** Verify identity of anyone accessing ePHI through passwords, MFA, or equivalent.

## 7. Payer Audit Requirements

### What Triggers an ABA Payer Audit

- **Outlier payments** (significantly above average reimbursement)
- **Higher-than-average utilization rates**
- **Increases in treatment or supervision hours**
- **Billing new procedure codes or locations**
- **Employee or patient complaints**
- **Previous audit errors/findings**
- **Random provider selection**

**Source:** [Little Health Law Blog - Preparing for a Payor Audit](https://www.littlehealthlawblog.com/preparing-your-aba-company-for-a-payor-audit/)

### What Payers Request During Audits

Payers typically request **primary source data** for a selected sample of patients and dates of service:

**Clinical Documentation:**

- Intake forms and initial assessments
- Functional Behavior Assessments (FBAs)
- Individualized Treatment Plans (ITPs) / Behavior Intervention Plans (BIPs)
- Session notes (both BCBA and RBT)
- Progress reports / periodic reviews
- Supervision notes and logs
- Discharge summaries (if applicable)

**Session Note Requirements (per session):**

- Client identifiers (full name, DOB)
- Date of service
- Exact start and end times (to the minute)
- Location of service
- CPT code and units billed (must match time documentation)
- Rendering provider name, credentials, and signature
- Targeted interventions linked to ITP goals
- Objective behavioral data (frequency, duration, ABC format)
- Progress indicators
- BCBA co-signature on RBT notes (within 48 hours)

**Provider Credentials:**

- BCBA certification (verified via BACB)
- Active state licensure (if required by state)
- NPI number
- Proof of RBT supervision compliance (5% monthly minimum)
- Liability insurance documentation
- CAQH profile

**Source:** [Praxis Notes - ABA Insurance Audit Documentation](https://www.praxisnotes.com/resources/aba-insurance-audit-documentation); [Praxis Notes - ABA Documentation Compliance Checklist](https://www.praxisnotes.com/resources/aba-documentation-compliance-checklist); [Praxis Notes - BCBA Audit Documentation Checklist](https://www.praxisnotes.com/resources/bcba-audit-documentation-checklist)

### Common Audit Findings/Failures

1. **Inadequate documentation**: Missing therapist or caregiver signatures, missing dates, incomplete session notes
2. **Billing irregularities**: Billing for group therapy incorrectly, overutilization of supervision hours, units billed don't match session times
3. **Missing or expired provider credentials**: Lapsed BCBA certification, expired state license, incomplete CAQH profile
4. **Improper billing codes**: Wrong CPT codes, missing modifiers, code-service mismatch

**Source:** [Little Health Law Blog - Preparing for a Payor Audit](https://www.littlehealthlawblog.com/preparing-your-aba-company-for-a-payor-audit/); [ABA Building Blocks - Clinical Documentation Compliance](https://ababuildingblocks.com/clinical-documentation-compliance/)

### Electronic Access & Audit Trail Requirements for Payer Audits

While payer audits primarily focus on **clinical documentation and billing accuracy**, not electronic access logs, there is increasing expectation that practices maintain:

- **Audit trails** showing who created/modified records and when
- **Evidence of HIPAA compliance** (role-based access controls, encryption, BAAs)
- **Credential verification records** proving provider qualifications at time of service
- **Supervision compliance logs** demonstrating 5% RBT supervision minimums
- **Record retention** for 7-10 years (varies by state and payer)

Software that logs every action (create, edit, view) with user ID and timestamp makes audit response significantly easier. The principle: **"If it isn't documented, regulators will treat it as if it doesn't exist."**

**Source:** [CubeTherapy - How to Keep ABA Documents Protected](https://www.cubetherapybilling.com/how-to-keep-your-aba-documents-protected-step-by-step-guide); [ABA Matrix - Audit-Proof Your ABA Practice](https://www.abamatrix.com/audit-proof-your-aba-practice-how-to-prepare-for-insurance-audits/)

## 8. Day-to-Day Workflows in a 1-50 Staff ABA Practice

### Typical Role Structure

| Role                                | Typical Count (20-person practice)          | Typical Count (50-person practice) |
| ----------------------------------- | ------------------------------------------- | ---------------------------------- |
| Practice Owner / Executive Director | 1                                           | 1                                  |
| Clinical Director (BCBA)            | 1 (often the owner)                         | 1-2                                |
| BCBAs (supervising)                 | 2-4                                         | 5-10                               |
| BCaBAs                              | 0-1                                         | 1-3                                |
| RBTs                                | 10-15                                       | 25-35                              |
| Office Manager / Admin              | 1                                           | 1-2                                |
| Billing Coordinator                 | 0-1 (often outsourced or shared with admin) | 1-2                                |
| Intake Coordinator                  | 0 (shared with admin)                       | 0-1                                |

**In small practices (1-20), staff are often cross-trained** - the office manager may handle billing, scheduling, and intake. The lead BCBA may also be the Clinical Director and practice owner.

**Source:** [ABA Building Blocks - Practice Management Guide](https://ababuildingblocks.com/aba-therapy-practice-management-tips/); [ABA Matrix - Administrative Roles in ABA](https://www.abamatrix.com/transitioning-from-clinical-to-administrative-roles-in-aba-how-to-make-the-shift/)

### Daily Workflows by Role

#### RBT Daily Workflow

1. **Morning:** Check schedule for today's sessions, review assigned client goals/protocols
2. **Travel/Arrive:** Travel to client location (home, clinic, school)
3. **Session prep:** Gather materials, review BIP, prepare data sheets
4. **Session delivery:** Implement treatment plan, collect behavioral data, engage caregiver
5. **Session documentation:** Write session notes (client ID, times, interventions, data, progress)
6. **Between sessions:** Travel, submit notes, check schedule changes
7. **End of day:** Ensure all session notes are submitted, review next day's schedule

**Software interactions:** View own schedule, view assigned client goals/BIP, enter session data, write session notes, clock time

#### BCaBA Daily Workflow

1. **Morning:** Review supervised RBTs' upcoming sessions, check own schedule
2. **Direct service:** Implement treatment plans for assigned clients (similar to RBT but with more clinical responsibility)
3. **RBT supervision:** Observe RBTs, provide feedback, document supervision
4. **Data review:** Analyze client data, flag concerns for supervising BCBA
5. **Documentation:** Write clinical notes, review/co-sign RBT session notes, document supervision contacts
6. **Coordination:** Communicate with BCBA about treatment modifications needed

**Software interactions:** View assigned clients' full records, view supervised RBTs' notes, enter supervision logs, write clinical notes, review data/graphs, co-sign RBT notes

#### BCBA Daily Workflow

1. **Morning:** Review caseload priorities, check authorization utilization, review flagged items
2. **Assessments:** Conduct FBAs, VB-MAPP, or other assessments for new/reassessing clients
3. **Treatment planning:** Develop/modify BIPs, set goals, update protocols
4. **Supervision:** Observe BCaBAs/RBTs, provide clinical guidance, document supervision (min 5% of RBT hours, 2 synchronous contacts/month, 1+ direct observation)
5. **Parent/caregiver training:** Conduct caregiver sessions (billed under 97156)
6. **Documentation:** Write assessment reports, progress reports, supervision notes, review/co-sign all supervised staff's notes within 48 hours
7. **Authorization management:** Review utilization, request new authorizations, submit clinical justifications
8. **Care coordination:** Consult with other providers (OT, SLP, school staff)

**Software interactions:** Full clinical access for assigned clients, create/edit treatment plans, review/co-sign all supervised staff notes, manage authorizations, view utilization dashboards, run clinical reports, document supervision, schedule management

#### Office Manager / Admin Daily Workflow

1. **Morning:** Review schedule for conflicts/gaps, handle incoming calls/intake inquiries
2. **Scheduling:** Coordinate client sessions, manage staff availability, handle cancellations/reschedules
3. **Intake:** Process new client referrals, collect demographics and insurance information
4. **Insurance verification:** Verify benefits, check eligibility, request initial authorizations
5. **Staff coordination:** Manage timesheets, coordinate onboarding, maintain credential files
6. **Communication:** Answer parent questions, coordinate with schools/facilities, send appointment reminders
7. **Compliance:** Maintain HIPAA documentation, manage consent forms, track credential expirations

**Software interactions:** All client demographics and scheduling, staff schedules, insurance information, authorization tracking (dates/units), credential management, intake forms -- NOT clinical notes or behavioral data

#### Billing Coordinator Daily Workflow

1. **Morning:** Review unbilled sessions, check claim status, process incoming payments/EOBs
2. **Claim submission:** Generate and submit claims for completed sessions (verify CPT codes, modifiers, units match session times)
3. **Payment posting:** Post ERA/EOB payments, reconcile with expected amounts
4. **Denial management:** Review denied claims, determine cause, resubmit or appeal
5. **Authorization monitoring:** Track utilization against approved units, alert clinical team of approaching limits, request re-authorizations
6. **Credentialing:** Maintain provider enrollment with payers, manage re-credentialing timelines
7. **Reporting:** Generate AR aging, collections, denial rate, and other financial reports

**Software interactions:** Session metadata (dates, times, codes, providers, units), client demographics and insurance, authorization details, claims and payment data, provider credentials, financial reports -- NOT clinical narrative content, behavioral data, or treatment plan details

## 9. RBAC Matrix Summary for Clinivise

Based on all regulatory requirements and workflow analysis above, here is the recommended permission matrix:

### Data Access by Role

| Data Category                            | RBT           | BCaBA           | BCBA          | Clinical Director | Office Admin | Billing Staff |
| ---------------------------------------- | ------------- | --------------- | ------------- | ----------------- | ------------ | ------------- |
| **Own schedule**                         | R             | R               | R             | R                 | RW           | R             |
| **All staff schedules**                  | -             | -               | -             | R                 | RW           | -             |
| **Assigned client demographics**         | R (limited)   | R               | R             | R                 | RW           | R             |
| **All client demographics**              | -             | -               | -             | R                 | RW           | R             |
| **Assigned client treatment plans/BIP**  | R             | R               | RW            | RW                | -            | -             |
| **All client treatment plans**           | -             | -               | -             | RW                | -            | -             |
| **Session notes (own)**                  | RW            | RW              | RW            | RW                | -            | -             |
| **Session notes (supervised staff)**     | -             | R (co-sign)     | R (co-sign)   | R (co-sign)       | -            | -             |
| **Session metadata (dates/times/codes)** | R (own)       | R (assigned)    | R (assigned)  | R (all)           | R (all)      | R (all)       |
| **Behavioral data/graphs**               | RW (assigned) | RW (assigned)   | RW (assigned) | R (all)           | -            | -             |
| **Assessment reports**                   | -             | R (assigned)    | RW            | RW                | -            | -             |
| **Insurance information**                | -             | -               | R (assigned)  | R                 | RW           | R             |
| **Authorization details**                | -             | R (units/dates) | RW            | RW                | R            | R             |
| **Claims & payment data**                | -             | -               | R (own NPI)   | R                 | -            | RW            |
| **Financial reports (AR, revenue)**      | -             | -               | -             | R                 | -            | RW            |
| **Provider credentials**                 | -             | -               | R (own)       | R (all)           | RW           | R             |
| **Supervision logs**                     | R (own)       | RW              | RW            | R (all)           | -            | -             |
| **Practice-wide clinical metrics**       | -             | -               | -             | R                 | -            | -             |
| **Audit trail / access logs**            | -             | -               | -             | R                 | R (admin)    | -             |

**Legend:** R = Read, W = Write, RW = Read/Write, - = No Access

### Key Regulatory Justifications

1. **RBTs see only assigned clients** -- HIPAA minimum necessary + RBT scope limited to supervised implementation
2. **Billing staff see session metadata but not clinical narratives** -- HIPAA minimum necessary for payment operations
3. **BCaBAs cannot independently modify treatment plans** -- BACB requires BCBA approval for modifications
4. **All session note edits logged with user ID and timestamp** -- 45 CFR 164.312(b) audit controls + payer audit requirements
5. **BCBA co-signature on RBT notes required** -- Payer audit requirement (typically within 48 hours)
6. **6-year minimum audit log retention** -- 45 CFR 164.316(b)(2)(i)
7. **7-year supervision documentation retention** -- BACB Ethics Code 4.05
8. **Unique user IDs, no shared accounts** -- 45 CFR 164.312(a)(2)(i) (required, not addressable)
9. **Automatic session timeout** -- 45 CFR 164.312(a)(2)(iii) (addressable but strongly recommended)
10. **Clinical Director needs practice-wide clinical access** -- Quality assurance and clinical oversight function
